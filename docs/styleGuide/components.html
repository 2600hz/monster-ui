Buttons

  • Structure
  • Roles & Styling

The base styling for buttons can be seen below—please note that there is a min-width and padding specified. This is to keep all buttons uniform, and these properties should not be changed. This base .btn class will never be used on its own, rather it will always be accompanied with a modifier class that specifies what Role & Styling the button obtains (as seen in the utility button below). There are currently two button structures: Base and Utility. Each can be seen below with their corresponding code. Note that utility buttons contain icons, implemented with <i> tags. Monster UI currently makes use of the Font Awesome icon library as well as a proprietary icon library known as Telicon.

<button class="btn">Base Style</button>

<button class="btn utility">
<i class="fa fa-cog">Utility Button
</button>
.btn {
font-size: 15px;
padding: 7px 20px;
min-width: 140px;
border: 1px solid;
border-radius: 2px;
position: relative;
letter-spacing: .02em;
}

.btn.utility {
background: transparent;
border-color: #22a5ff;
color: #22a5ff;
}
.btn.utility i {
margin-right: 8px;
}

There are currently six different levels of button hierarchy, with each button having a specific role and style.

Primary

Primary buttons are only to be used when there is a clear call-to-action that the user needs to take. A few examples are actions such as Add Credit, Change Info, or Update Limit.

View Class Styles
.btn.primary {
background: #22a5ff;
border-color: #0087e0;
color: #004c80;
}

Utility / Functionality

Utility buttons signify secondary actions that usually address another element. Some examples are actions like Refresh, Filter, or Download.

View Class Styles
.btn.utility {
background: transparent;
border-color: #22a5ff;
color: #22a5ff;
}

Confirmation / Finalization

These buttons are ONLY used for final confirmation of actions, and almost solely live within confirmation dialogue modals. Confirmations such as Save Changes or Purchase fall under this role.

View Class Styles
.btn.confirm {
background: #33DB24;
border-color: #18b309;
color: #0e6b06;
}

Cancel / Undo / Disregard Changes

These buttons are used to cancel actions/ changes before a Confirming. As such, they are almost always paired with a Confirmation button.

View Class Styles
.btn.cancel {
background: transparent;
border-color: #c0c0c9;
color: #909099;
}

Warning Implications

Used very sparingly, these are used for actions that can alter serious funcitonality of the platform, but are most likely temporary changes. An example would be to Deactivate an account until an overdue balance is paid.

View Class Styles
.btn.warning {
background: #ffcc33;
border-color: #f0b400;
color: #8f6b00;
}

Removal / Finalization

This role is pretty self-explanatory, and is ONLY used as a final confirmation to delete/ remove something from the system.

View Class Styles
.btn.delete {
background: #ff3d24;
border-color: #e01a00;
color: #8a1000;
}
